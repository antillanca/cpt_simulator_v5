## 2. Current State & Core Capabilities (V2.9E)

The codebase currently implements the following advanced features:

1. **Topology Curriculum (`backend/circuits/topology_curriculum.py`)**: 
   Circuits are dynamically classified into 4 complexity levels:
   - **Level 0 (Trivial)**: Tree-like, $\le 4$ nodes, 0 cycles.
   - **Level 1 (Simple)**: 1 Cycle, $\le 6$ nodes.
   - **Level 2 (Medium)**: 2-3 Cycles, $\le 10$ nodes.
   - **Level 3 (Dense)**: $>3$ Cycles, $> 10$ nodes.
   *This curriculum scheduler progressively unlocks harder topologies during training.*

2. **Graph Feature Enrichment & Normalization (`backend/circuits/graph_dataset.py`)**:
   - Node and edge dimensions scale dynamically.
   - Edges include cycle participation flags and structural data.
   - Resistance values are **log-normalized** to prevent gradient explosions in extreme ranges.

3. **Topology-Aware Failure Taxonomy (`backend/circuits/failure_analysis.py`)**:
   The diagnostic engine now classifies physical violations into sharp root-causes:
   - `cycle_drift_failure`: KCL drift within closed loops.
   - `dense_mesh_leakage`: Signal dissipation in high-degree interconnected meshes.
   - `bridge_node_instability`: Drift in bottleneck/cut-vertex nodes in trees.

4. **OOD Stress Suite (`backend/circuits/ood_stress_suite.py`)**:
   Deterministic generators for pathologically hard circuits to benchmark the GNN:
   - `generate_dense_grid`: Massive grids testing KCL constraints.
   - `generate_ladder_network`: Long chains testing GNN receptive fields.
   - `generate_cycle_dominant_loops`: Interlocking rings.
